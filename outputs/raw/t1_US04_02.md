### US04-CT01 — Desativar usuário ativo com sucesso
- **Pré-condição:** Admin autenticado na tela de gestão de usuários. Usuário (não o próprio admin) com status "Ativo" existe no sistema.
- **Objetivo:** Verificar desativação de usuário ativo e remoção de acesso.
- **Resultado esperado:** Status do usuário altera para 'INATIVO'. Login do usuário desativado é negado com mensagem de conta inativa.
- **Tipo:** Principal
- **Critérios cobertos:** Cenário 1
- **Verificação:** SUPORTADO — evidência: Admin pode desativar qualquer usuário, alterando status para INATIVO. Usuário inativo não consegue autenticar. — fonte: docs/entrega-sprint-1.md, docs/criterios-de-aceitacao.md, docs/sprints/sprint-1.md (tipo: documentação)
- **Origem:** A1
- **Aprovado humano:** pendente

### US04-CT02 — Tentativa de desativar usuário já inativo
- **Pré-condição:** Admin autenticado na tela de gestão de usuários. Usuário com status "Inativo" já existe no sistema.
- **Objetivo:** Verificar sistema ao tentar desativar usuário já inativo.
- **Resultado esperado:** Sistema retorna erro: usuário já inativo. Status do usuário permanece 'INATIVO'.
- **Tipo:** Exceção
- **Critérios cobertos:** Cenário 2
- **Verificação:** SUPORTADO — evidência: Requisição para desativar usuário já desativado retorna erro. — fonte: user_stories/individual/us04.md (tipo: us)
- **Origem:** A1
- **Aprovado humano:** pendente

### US04-CT03 — Tentativa de desativar o próprio usuário admin
- **Pré-condição:** Admin autenticado na tela de gestão de usuários.
- **Objetivo:** Verificar impedimento do admin desativar a si mesmo.
- **Resultado esperado:** Sistema impede desativação do admin por ele mesmo, exibe erro. Status do admin permanece 'Ativo'.
- **Tipo:** Exceção
- **Critérios cobertos:** Validação: admin não pode desativar a si mesmo (previne lock-out)
- **Verificação:** SUPORTADO — evidência: Admin não pode desativar a si mesmo (previne lock-out). — fonte: docs/sprints/sprint-1.md (tipo: documentação)
