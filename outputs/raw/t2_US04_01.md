### US04-CT01 — Desativar usuário ativo com sucesso
- **Pré-condição:** Admin autenticado na tela de gestão de usuários; existe pelo menos um usuário com status ATIVO (ou PENDENTE) cujo ID não seja o do admin logado.
- **Objetivo:** Validar que desativar usuário ativo altera status para INATIVO e bloqueia acesso ao sistema.
- **Resultado esperado:** Endpoint retorna sucesso; status atualizado para INATIVO; login subsequente falha.
- **Tipo:** principal
- **Critérios cobertos:** AC1
- **Verificação:** SUPORTADO — evidência: Admin pode desativar qualquer usuário, alterando status para INATIVO. Usuário inativo não consegue autenticar — verificação feita no fluxo de login. — fonte: docs/entrega-sprint-1.md#___US04___Admin_desativa_um_usu_rio (tipo: documentação)
- **Origem:** A1 (US04) + docs/entrega-sprint-1
- **Aprovado humano:** pendente
