### US04-CT01 — Desativa usuário ativo
- **Pré-condição:** Admin autenticado na tela de gestão de usuários; existe usuário com status ATIVO.  
- **Objetivo:** Validar que admin pode desativar usuário ativo.  
- **Resultado esperado:** Status muda para INATIVO e exibe mensagem de sucesso.  
- **Tipo:** principal  
- **Critérios cobertos:** AC1 (Cenário 1: Desativa usuário ativo)  
- **Verificação:** SUPORTADO — evidência: Admin pode desativar qualquer usuário, alterando status para INATIVO; cenário 1 confirma que o acesso é desativado — fonte: docs/entrega-sprint-1.md#___US04___Admin_desativa_um_usu_rio (tipo: documentação); user_stories/individual/us04.md (tipo: us)  
- **Origem:** A1  
- **Aprovado humano:** pendente

### US04-CT03 — Tentativa de autodesativação (admin tenta desativar a si mesmo)

- **Objetivo:** Validar que sistema impede admin de se desativar a si mesmo (prevenção de lock‑out).  
- **Resultado esperado:** Sistema retorna erro indicando que não é permitido desativar próprio usuário e mantém status ATIVO.  
- **Tipo:** excepcional  
- **Critérios cobertos:** a confirmar (regra de não auto‑desativação mencionada em docs/sprints/sprint‑1.md)

- **Objetivo:** Validar que usuário desativado não pode autenticar no sistema.  
- **Resultado esperado:** Ao tentar login com credenciais válidas, sistema retorna erro de conta inativa e nega acesso.
