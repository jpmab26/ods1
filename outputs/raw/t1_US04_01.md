### US04-CT01 — Desativação de usuário ativo
- **Pré-condição:** Admin autenticado na tela de gestão de usuários. Existe um usuário ativo (Aluno, Monitor ou Professor) que não é o próprio admin.
- **Objetivo:** Verificar se o admin consegue desativar um usuário ativo e remover seu acesso ao sistema.
- **Resultado esperado:** O status do usuário é "INATIVO". O usuário desativado não consegue mais fazer login e o admin recebe uma mensagem de sucesso.
- **Tipo:** principal
- **Critérios cobertos:** Cenário 1: Desativa usuário ativo, Cenário 4 (US02): Usuário inativo não loga
- **Verificação:** SUPORTADO — evidência: Admin desativa, status "INATIVO", flash "Usuário desativado com sucesso", login negado para inativos. — fonte: docs/entrega-sprint-1.md (tipo: documentação), docs/artefatos-qscrum-sprint-1.md (tipo: documentação), docs/sprints/sprint-1.md (tipo: documentação), backend/usuarios/routes.py (tipo: código), docs/criterios-de-aceitacao.md (tipo: documentação)
- **Origem:** US04, docs/entrega-sprint-1.md, docs/artefatos-qscrum-sprint-1.md, docs/criterios-de-aceitacao.md (US02-Cenário 4), docs/sprints/sprint-1.md
- **Aprovado humano:** pendente
